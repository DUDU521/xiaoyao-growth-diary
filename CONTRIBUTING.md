# 贡献指南

欢迎贡献你的经验和教训！以下是贡献的具体步骤：

## 📝 内容格式要求

### 文件命名
- 使用英文小写字母和连字符：`example-experience.md`
- 避免特殊字符和空格

### 文档结构
```markdown
# 标题

## 问题描述
清晰描述遇到的问题或场景

## 解决方案
详细说明解决方案的步骤

## 核心洞察
提炼出的关键经验和原则

## 适用场景
说明这个经验适用的情况

## 注意事项
可能的陷阱和预防措施

## 相关链接
相关的工具、文档或资源
```

### 元数据
在文件开头添加Front Matter：
```yaml
---
date: 2026-03-19
tags: [ai, automation, learning]
category: pitfalls
difficulty: medium
---
```

## 🚀 贡献流程

1. **Fork项目**到你的GitHub账户
2. **创建新分支**：`git checkout -b feature/your-experience`
3. **添加内容**到对应的分类目录
4. **提交更改**：`git commit -m "Add experience: your title"`
5. **推送分支**：`git push origin feature/your-experience`
6. **创建Pull Request**

## 📊 分类说明

- **🐎 逍遥日志**：日常记录、工作日志
- **📝 方法论**：系统性方法、工作流程
- **🔗 工具发现**：新工具、新技术栈
- **💡 灵感闪现**：创意想法、创新思路  
- **🧠 认知升级**：思维转变、理解深化
- **🗂️ 踩坑记录**：问题分析、错误解决
