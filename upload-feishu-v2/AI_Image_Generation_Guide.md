## 📝 提示词编写技巧

### 好的提示词特征

✅ **具体描述**: "夕阳下的海滩，金色的沙滩，蓝色的海水，海鸥飞翔"  
✅ **风格指定**: "科技感，蓝色调，扁平化设计"  
✅ **元素明确**: "包含3个模块：用户界面、数据处理、结果展示"  
✅ **色彩搭配**: "绿色健康色调，清新自然"

❌ **模糊描述**: "一张好看的图片"  
❌ **过于抽象**: "美丽的风景"  
❌ **缺少细节**: "一个系统界面"

### 提示词模板

```
[主体] + [动作/状态] + [环境/背景] + [风格/色调] + [特殊要求]

示例:
"智慧医院系统界面(主体) + 数据可视化(动作) + 现代化医疗环境(背景) + 科技蓝(色调) + 包含大屏仪表盘(特殊要求)"
```

## ⚙️ 配置说明

### 配置文件

编辑 `config.json`:

```json
{
  "api_key": "your-modelscope-token",
  "model": "Tongyi-MAI/Z-Image-Turbo",
  "output_dir": "./generated_images",
  "timeout": 300,
  "poll_interval": 5
}
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `api_key` | ModelScope API Token | 已配置 |
| `model` | 使用的生成模型 | Tongyi-MAI/Z-Image-Turbo |
| `output_dir` | 输出目录 | ./generated_images |
| `timeout` | 最大等待时间（秒） | 300 |
| `poll_interval` | 轮询间隔（秒） | 5 |

## 📋 批量生成

### 批量生成脚本

创建 `batch_generate.sh`:

```bash
#!/bin/bash

prompts=(
  "互联网医院推广流程图，商务风格，蓝绿色调"
  "医疗闭环数据整改流程图，专业医疗风格"
  "出院智能体工作界面，现代化医院环境"
  "办公技能安装界面截图，简洁商务风"
  "技能清单总览图，信息图表风格"
)

for i in "${!prompts[@]}"; do
  echo "生成图片 $((i+1))/${#prompts[@]}..."
  python3 scripts/generate.py "${prompts[$i]}" "batch_image_$i.jpg"
done

echo "✅ 批量生成完成！"
```

### 执行批量生成

```bash
chmod +x batch_generate.sh
./batch_generate.sh
```

## 🔧 故障排查

### 问题1：生成失败

**错误信息**: "Image Generation Failed"

**解决方案**:
1. 检查网络连接
2. 验证API Key是否有效
3. 检查提示词是否包含敏感内容

### 问题2：生成超时

**错误信息**: "生成超时"

**解决方案**:
1. 增加 `timeout` 参数（建议300-600秒）
2. 简化提示词
3. 检查ModelScope服务状态

### 问题3：图片质量不佳

**解决方案**:
1. 优化提示词，增加细节描述
2. 尝试不同风格关键词
3. 多次生成选择最佳结果

## 📊 最佳实践

### 1. 提示词迭代

第一次生成后，根据结果调整提示词：

```
初版："系统界面"
↓
二版："科技感系统界面，蓝色调"
↓
终版："现代化系统界面，科技蓝，包含数据面板、用户列表、操作按钮三个区域"
```

### 2. 风格统一

为同一项目生成图片时，保持风格一致：

```json
{
  "统一色调": "科技蓝",
  "统一风格": "扁平化",
  "统一元素": "圆角矩形、渐变背景"
}
```

建立图片索引和命名规范：

```

├── README.md              # 图片索引
├── architecture_*.jpg     # 架构图系列
├── flowchart_*.jpg        # 流程图系列
└── interface_*.jpg        # 界面图系列
```
