## 🎯 角色详解

### 1️⃣ 皇帝（用户）- 高中涛

**职责**:
- 下达战略意图（如："查一下于卫芳的电话"）
- 审批重要决策（如："安装这个技能"）
- 掌控帝国方向（如："整理临床路径需求"）

**特点**:
- 不需要懂技术细节
- 用自然语言表达需求
- 拥有最高决策权

**示例旨意**:
```
"帮我查一下于卫芳的电话"
"把这份Excel整理成Markdown"
"明天凌晨1点执行签到任务"
```

**职责**:
- 理解皇帝意图（NLU）

---

### 5️⃣ 军令牌（MCP - Model Context Protocol）

**核心作用**:
统一不同军师（大模型）的调兵格式

**标准化内容**:
```json
{
  "tool": "web_search",
  "parameters": {
    "query": "于卫芳 河北医科大学第一医院",
    "count": 5
  }
}
```

**意义**:
- 不同军师（Claude、GPT、Kimi）使用同一套指令
- 技能可以跨军师复用
- 降低开发成本

---

### 6️⃣ 军事手册（Skill）

**定义**:
告诉军师"何时做、如何做、按什么顺序做"

**结构**:
```yaml
name: excel-parser
description: 专业的Excel文件解析
triggers: ["excel", "表格", "xlsx"]  # 何时调用
steps:                                    # 如何做
  - 读取Excel文件
  - 搜索关键词
  - 返回结果
```

**示例技能**:
| 技能 | 功能 | 触发词 |
|------|------|--------|
| excel-parser | Excel解析 | excel, 表格 |
| markitdown | 文档转换 | markdown, 转换 |
| checkin | 自动签到 | 签到, checkin |
| tavily | 网络搜索 | 搜索, 查资料 |

### 7️⃣ 将军/士兵（Tools）

**基础工具集**:

| 工具 | 角色 | 功能 |
|------|------|------|
| **read** | 文书官 | 读取文件内容 |
| **write** | 史官 | 写入文件 |
| **edit** | 校书郎 | 修改文件 |
| **exec** | 武将 | 执行命令 |
| **web_search** | 探子 | 网络搜索 |

| **browser** | 密探 | 浏览器操作 |
| **message** | 信使 | 发送消息 |

**工具调用示例**:
```python
# 文书官读文件
read(file_path="电话表.xlsx")

# 武将执行命令
exec(command="python search.py 电话表.xlsx 于卫芳")

# 探子网络搜索
web_search(query="OpenClaw skills excel")
```

## 🎨 架构优势

### 1. 分层解耦
- 皇帝不用懂技术
- 军师不用懂具体实现
- 技能开发者不用懂AI

### 2. 模块化扩展
- 新增技能 = 新增军事手册
- 新增工具 = 新增兵种
- 不影响现有体系

### 3. 标准化接口
- MCP统一协议
- 跨模型兼容
- 降低迁移成本

### 4. 生态繁荣
- ClawHub技能市场
- 社区共建共享
- 持续进化
