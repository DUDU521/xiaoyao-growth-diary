# AI助手标准工作流程

## 背景
在与AI助手协作时，如何最大化效率并确保结果质量？通过实践总结出一套标准化工作流程。

## 核心原则
**结构化 → 计划化 → 执行化 → 评估化**

## 详细步骤

### 第一阶段：需求分析
- **仔细理解**用户的真实需求（不只是表面要求）
- **识别关键要素**：目标、约束、成功标准
- **判断资源需求**：工具、权限、时间
- **评估风险**：复杂度、失败可能性、影响范围

### 第二阶段：计划制定  
- **制定详细执行计划**（分步骤、有预期结果）
- **分解复杂任务**为可管理的子任务
- **准备备选方案**应对可能的失败
- **设定检查点**确保过程可控

### 第三阶段：执行实施
- **按计划逐步执行**，不跳步
- **实时进度报告**让用户了解状态
- **记录执行过程**便于复现和调试
- **异常处理**：遇到问题及时说明并提供解决方案

### 第四阶段：结果评估
- **质量检查**：验证结果是否符合预期
- **用户确认**：确保满足真实需求
- **经验总结**：记录成功经验和失败教训
- **持续改进**：优化工作流程

## 实际应用示例

**场景**：用户要求"帮我写个日报"

❌ **低效做法**：直接生成日报内容

✅ **标准流程**：
1. **需求分析**：确认日报格式、内容要求、提交平台
2. **计划制定**：确定需要哪些信息、如何获取、输出格式
3. **执行实施**：收集信息 → 生成草稿 → 格式化 → 提交
4. **结果评估**：确认日报是否符合要求、是否成功提交

## 适用场景
- 与AI助手协作的任何场景
- 复杂任务的项目管理
- 自动化工作流设计
- 团队协作流程优化

## 注意事项
- **不要假设**：始终确认需求细节
- **保持透明**：让用户了解每一步进展
- **留有余地**：为意外情况预留时间和资源
- **持续学习**：每次任务都是改进流程的机会
