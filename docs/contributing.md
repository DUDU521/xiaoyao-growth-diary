# 如何贡献

欢迎任何人分享自己的经验和教训！这个项目的价值在于汇集更多人的智慧。

## 📝 为什么贡献？

- **帮助他人**：你的经验可能正是别人需要的
- **完善自己**：整理经验的过程也是深度思考的过程  
- **建立连接**：通过分享找到志同道合的伙伴
- **持续成长**：在交流中获得新的启发

## 🚀 贡献方式

### 1. 直接提交PR（推荐）
如果你熟悉Git工作流，可以直接按照[贡献指南](/CONTRIBUTING.md)提交Pull Request。

### 2. 提交Issue
如果你不熟悉Git，可以：
- 在Issues中描述你的经验
- 我会帮你格式化并添加到项目中
- 保持原作者署名

### 3. 联系作者
通过其他渠道（如邮件、社交媒体）分享，我会整理后加入。

## 🎯 内容建议

好的经验分享应该包含：

✅ **具体场景** - 在什么情况下遇到的问题  
✅ **详细过程** - 如何分析和解决问题  
✅ **核心原则** - 提炼出的通用方法论  
✅ **适用边界** - 什么情况下适用，什么情况下不适用  
✅ **后续验证** - 解决方案是否真的有效

避免：

❌ 纯理论空谈  
❌ 缺乏具体细节  
❌ 只说结论不说过程

## 💬 社区准则

- **尊重他人**：不同观点要友善讨论
- **事实为准**：基于实际经验，不编造内容  
- **开放心态**：接受建设性批评和改进建议
- **持续维护**：如果发现错误，请及时指出

---

**记住：每一次分享，都是在为后来者点亮一盏灯。**
